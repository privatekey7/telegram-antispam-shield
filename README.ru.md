# 🛡 Telegram Anti-Spam Shield

[English](README.md) · 🌍 **Русский**

Лёгкий Telegram-бот, который **автоматически находит и удаляет спам** (на русском
и английском) с помощью маленькой встроенной ML-модели. Он быстрый, бесплатный
в хостинге и сделан так, чтобы **любой мог развернуть его за ~10 минут** — без
программирования.

## Что он умеет

- 🤖 Классифицирует каждое сообщение в группе как **спам** или **не спам**.
- 🗑 **Удаляет** спам автоматически (а ещё может **банить** или **мьютить** отправителя).
- 🇷🇺 🇬🇧 Обучен на реальном русском и английском спаме.
- 🪶 Крошечный расход ресурсов: десятки МБ RAM — влезает в бесплатные тарифы.
- 🔌 Без внешних AI-API и без оплаты за сообщение. Модель **встроена в бота**.
- 🛟 Безопасен по умолчанию: админов не трогает, а **режим dry-run** позволяет
  посмотреть, как он работает, прежде чем он начнёт что-то удалять.

### Насколько он хорош?

Замерено на отложенной тестовой выборке из публичных спам-данных:

| Метрика | Значение |
|---|---|
| Точность (accuracy) | **97.0%** |
| ROC-AUC | **0.994** |
| Поймано спама (recall) | **93.6%** |
| Доля ложных срабатываний (удалено хорошее) | **~1.0%** |
| Размер модели | **4.2 МБ** (встроена) |

Порог принятия решения специально настроен так, чтобы **почти никогда не удалять
нормальные сообщения** — лучше пропустить часть спама, чем удалить настоящее
сообщение.

## 🚀 Развёртывание за 10 минут (без программирования)

Вы сделаете три вещи: (1) создадите бота, (2) разместите проект на Railway,
(3) добавите бота в группу. Выполняйте шаги по порядку.

### Шаг 1 — Создайте бота и получите токен

1. В Telegram откройте [@BotFather](https://t.me/BotFather).
2. Отправьте `/newbot` и следуйте подсказкам (придумайте имя и username).
3. BotFather выдаст **токен** вида
   `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`. **Скопируйте его** — он понадобится.
4. Отправьте `/setprivacy` → выберите своего бота → нажмите **Disable**. Это
   разрешит боту видеть все сообщения в группе и проверять их на спам. *(Важно!)*

### Шаг 2 — Разместите бота на Railway

1. **Сделайте форк** этого репозитория (кнопка **Fork** вверху справа на GitHub).
2. Зайдите на [railway.com](https://railway.com) и войдите через GitHub.
3. Нажмите **New Project** → **Deploy from GitHub repo** → выберите свой форк.
4. Railway увидит готовый `Dockerfile` и начнёт сборку автоматически.
5. Откройте вкладку **Variables** и добавьте одну переменную:
   - Имя: `BOT_TOKEN`
   - Значение: токен из Шага 1
6. Railway передеплоит. Когда в **Deploy Logs** появится `bot_started` — бот
   запущен. 🎉

> 💡 Для пробного/бесплатного использования Railway карта не нужна. Бот почти не
> потребляет ресурсов.

### Шаг 3 — Добавьте бота в группу

1. Откройте вашу Telegram-**группу** → **Добавить участников** → добавьте бота.
2. Сделайте бота **администратором** как минимум с правом **Удалять сообщения**.
   *(Чтобы ещё банить/мьютить спамеров — включите также **Блокировка пользователей**.)*
3. Готово. Бот теперь удаляет спам автоматически.

**Проверка:** отправьте в группе что-нибудь спамное, например
`ЗАРАБОТОК ОТ 5000$ В ДЕНЬ! Пиши в личку` — сообщение должно удалиться за секунду.

> 🧪 Хотите сначала понаблюдать? Поставьте `ANTISPAM_DRY_RUN=true` в Variables на
> Railway. Бот будет только писать в лог, что он *удалил бы*. Верните `false`,
> когда убедитесь, что всё хорошо.

## ⚙️ Настройки

Все настройки — это **переменные окружения** (задаются в Railway → Variables).
Обязательна только `BOT_TOKEN`, у всего остального есть разумные значения по умолчанию.

| Переменная | По умолчанию | Описание |
|---|---|---|
| `BOT_TOKEN` | — | **Обязательно.** Токен от @BotFather. |
| `ANTISPAM_ACTION` | `delete` | `delete`, `ban` или `mute` для спамера. |
| `ANTISPAM_DRY_RUN` | `false` | Если `true` — только логировать, ничего не удалять. |
| `ANTISPAM_THRESHOLD` | *(из модели ≈ 0.82)* | Порог вероятности спама `0.0–1.0`. Выше = строже (меньше удалений). |
| `ANTISPAM_BLOCK_INVITE_LINKS` | `true` | Считать ссылки `t.me/+…` / `joinchat` спамом. |
| `ANTISPAM_EXEMPT_ADMINS` | `true` | Никогда не модерировать админов группы. |
| `ANTISPAM_ADMIN_CACHE_TTL` | `3600` | Сколько секунд кэшировать список админов группы. |
| `RUST_LOG` | `info` | Уровень логов: `error`/`warn`/`info`/`debug`. |
| `MODEL_PATH` | *(не задано)* | Путь к своей обученной модели (для продвинутых). |

Шаблон для копирования — в файле [`.env.example`](.env.example).

## 🧠 Как это работает

```
Входящее сообщение
   │
   ├─ 1. Эвристики → инвайт-ссылки и т.п. (мгновенно, высокая точность) → спам → удалить
   │
   ├─ 2. ML-модель → вероятность спама ≥ порога?                        → спам → удалить
   │
   └─ иначе: пропустить
```

- **Извлечение признаков** превращает текст в хешированные словные и символьные
  n-граммы (`src/features.rs`). Символьные n-граммы дают устойчивость к
  обфускации и работают для разных языков.
- **Модель** — логистическая регрессия, хранится как бинарный файл 4 МБ, вшита в
  исполняемый файл, считается на чистом Rust (`src/model.rs`).
- **Обучение** происходит офлайн на Python (`scripts/train_model.py`). Тест
  совместимости (`tests/parity.rs`) гарантирует, что Rust-инференс совпадает с
  Python-обучением.

Ключевые архитектурные решения записаны в [`docs/adr/`](docs/adr/).

## 🛠 Разработка

```bash
# Тесты, линтер и форматтер (нужен Rust stable)
cargo test
cargo clippy --all-targets -- -D warnings
cargo fmt --check

# Локальный запуск (читает BOT_TOKEN из локального .env)
cp .env.example .env   # затем впишите токен в .env
cargo run
```

## 🔁 Переобучение модели (опционально)

Бот поставляется с готовой моделью, поэтому это нужно только продвинутым пользователям.

```bash
pip install -r scripts/requirements.txt
python scripts/train_model.py     # скачает датасеты, обучит, запишет model/spam_model.bin
cargo test                        # parity-тест подтвердит, что Rust совпадает с новой моделью
```

Используемые датасеты:
[`alt-gnome/telegram-spam`](https://huggingface.co/datasets/alt-gnome/telegram-spam)
и [`mshenoda/spam-messages`](https://huggingface.co/datasets/mshenoda/spam-messages).

## ❓ Решение проблем

| Симптом | Что сделать |
|---|---|
| Бот ничего не удаляет | Сделайте его **админом** группы с правом **Удалять сообщения**; отключите приватность через `/setprivacy` в BotFather. |
| В логах `delete_failed` | У бота нет прав на удаление — поправьте права администратора. |
| Удалил нормальное сообщение | Поднимите `ANTISPAM_THRESHOLD` (например, `0.9`) или сообщите о примере в issue. |
| Хочу протестировать безопасно | Поставьте `ANTISPAM_DRY_RUN=true`. |
| Сборка падает на Railway | Убедитесь, что деплоите свой **форк** без изменений; Railway должен использовать `Dockerfile`. |
